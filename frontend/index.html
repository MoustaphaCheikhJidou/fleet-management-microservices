Pilotage temps réel

Un cockpit web unique pour orchestrer véhicules, conducteurs et incidents.

Chaque requête transite par le Gateway sécurisé, les services récupèrent leur configuration via Config Server et se fédèrent dans Eureka. Les rôles sont portés par un JWT unique que l’Admin Portal peut gouverner sans scripts.

Trame cloud-native

Build Maven unique, services Spring Boot isolés avec Dockerfile dédié. Les pipelines utilisent la commande ./mvnw -DskipTests package avant tout déploiement compose.

Résilience opérationnelle

Resilience4J protège les appels Vehicle/Driver/Shipments, tandis que RabbitMQ assure la diffusion d’événements IAM vers les services aval.

Expérience admin

Un template responsive permet aux équipes support de créer, activer ou suspendre un compte sans outils externes tout en respectant la charte de sécurité IAM.

Mission

Application de suivi de flotte distribuée

Exposer l’état complet d’un véhicule (conducteur, position, incidents) en orchestrant des microservices Spring.

Vehicle Service Driver Service Tracking Service Incident Service

Technologies imposées

  • Eureka pour la découverte
  • Feign & Resilience4J pour la communication
  • Gateway + RoleAuthorizationFilter pour la sécurité
  • Config Server + Actuator pour l’observabilité

Livrables clés

  • Seed SuperAdmin configurable via SUPERADMIN_* env vars
  • Console d’authentification dédiée (login & sign-up)
  • Admin Portal pour la gestion des comptes et rôles
  • Scripts de test (test_endpoints.py / test_apis.sh)
